# MOMO

───────────────────────────────
📘 프로그램명 : MOMO (농산물 주문 자동화 앱)
───────────────────────────────

[1] 개발환경

-- 프론트엔드 : React Native (Expo)

-- 백엔드 : Node.js (Express)

-- 데이터베이스 : MySQL

-- 문자 발송 API : CoolSMS

───────────────────────────────

[2] 주요 기능

-- 회원가입 (휴대전화 인증번호 기반)

-- 로그인 / 자동 로그인 (AsyncStorage)

-- 상품 등록, 수정, 삭제 (관리자 전용)

-- 상품 목록 조회 및 상세보기

-- 장바구니 담기 / 바로구매 기능

-- 배송지 입력 및 개인정보 수집 동의 절차

-- 주문번호 자동 생성 (YYYYMMDD + 4자리)

-- 결제(계좌이체) 후 ‘입금 완료’ 버튼 클릭 시 관리자에게 문자 자동 발송

-- 관리자 주문상태 변경 (입금대기 → 배송중 → 배송완료)

-- 주문상태 변경 시 문자 자동 발송

-- 입금대기 상태 2일 유지 후, 3일째 자동 취소 처리 (cron 스케줄러 작동)

-- 주문 후 1년 경과 시 개인정보 자동 삭제

───────────────────────────────

[3] 실행 환경

Android (APK 설치파일 포함)

iOS : 추후 대응 예정

실행 방법

app 폴더 : React Native 프로젝트
→ npm install
→ npx expo start
→ Android 기기 또는 Expo Go 앱에서 실행 가능

별도의 서버 설정 불필요
→ AWS 환경에서 이미 서버가 동작 중이므로 앱 실행 시 자동 연결됨
※ .env 파일 등 서버 환경 정보는 보안상 포함하지 않음

───────────────────────────────

[4] 서버 정보

본 앱은 AWS(EC2) 상에서 동작 중인 Node.js 서버와 연동되어 있습니다.

앱 내부의 fetch / axios 요청을 통해 API 호출 구조를 확인할 수 있습니다.

주요 엔드포인트
/api/products, /api/orders, /api/send-auth-code, /api/update-order-status 등

[6] 비고

본 소스코드는 저작권 등록용 복제물이며,
서버 접속정보(.env) 및 API 인증키는 포함하지 않았습니다.

AWS 서버와 연동된 실제 서비스형 구조로,

본 복제물만으로도 프로그램의 주요 기능과 작동 원리를 완전히 확인할 수 있습니다.

CoolSMS API의 서비스 잔액(지갑 금액)이 소진되어, 문자 발송 기능은 실제 실행 시 제한됩니다.
